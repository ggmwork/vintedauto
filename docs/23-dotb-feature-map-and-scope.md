# Dotb Feature Map And Scope

Last updated: 2026-04-28

## Purpose

This document translates the useful parts of Dotb into a safer product scope for this app.

## Build now

These are high-value and fit the current safety boundary.

### 1. Vinted web autofill extension

- fill title
- fill description
- fill price
- fill category and item metadata where possible
- upload images
- require manual review and manual submit

### 2. Listing dashboard and review queue

- draft queue
- review and save
- copy and next
- fill on Vinted and next

### 3. Stock and inventory management

- SKU
- storage location
- status
- cover image
- target account

### 4. Internal multi-account management

- multiple account profiles
- per-account listing queue
- per-account presets
- per-account stock assignment
- per-account performance views

### 5. Orders and sales tracking

- order records
- sale price
- sold date
- buyer country
- account used
- fulfillment status

### 6. Profit and accounting basics

- cost basis
- packaging cost
- net profit
- margin
- monthly and per-account summaries
- CSV export

### 7. Quick replies and templates

- reusable description presets
- reusable chat text snippets
- account-specific templates

## Build later

These are useful, but should follow the core extension and admin workflow.

### 1. Shipping label helpers

- label crop
- thermal formatting
- label annotations

### 2. Order panel upgrades

- advanced filters
- bulk review actions
- deadline reminders

### 3. Accounting views

- quarterly rollups
- account comparisons
- export presets

### 4. Google Sheets / CSV sync

- export first
- import later only if needed

### 5. Conversation workspace

- inbox panel
- quick reply insertion

## Defer or avoid

These are intentionally not next.

### High-risk automation

- auto messages to likes
- auto negotiation
- auto accept offers
- auto reposting
- restocker automation

### High-risk cross-account behavior

- cross-account listing import
- duplicate listing propagation
- aggressive multi-account growth workflows

### Broader platform expansion

- multi-marketplace support
- mobile app automation
- full ERP / tax system

## Guidance

When deciding whether to build a Dotb-like feature, ask:

1. does it help the seller prepare or publish faster on Vinted web?
2. does it keep the human in control?
3. does it improve internal operations, stock, orders, or profit visibility?
4. does it avoid high-risk platform behavior?

If not, it is not a priority.
