# Karpathy Guidelines

Think before coding:

- State assumptions when they matter.
- Ask instead of guessing when ambiguity changes implementation.
- Surface tradeoffs and push back on unnecessary complexity.

Simplicity first:

- Write minimum code that solves requested problem.
- No speculative abstractions or extra features.

Surgical changes:

- Touch only lines needed for task.
- Match existing style.
- Do not refactor adjacent code unless task requires it.

Goal-driven execution:

- Define clear verification for each step.
- Keep working until result is checked.
